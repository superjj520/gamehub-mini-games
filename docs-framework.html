5. 游戏配置化操作指南 ?c= 参数
6个抽奖游戏支持通过链接传入自定义配置，实现一个游戏无限玩法
📐 原理：链接即配置

配置是一个 JSON 对象，经过 Base64 编码后附加到游戏 URL 的 ?c= 参数。
玩家打开链接时，游戏自动读取配置并渲染，不同的链接就是不同的活动。

// 配置对象结构
{
  "title": "双11大转盘", // 游戏标题
  "subtitle": "每天3次免费抽", // 副标题
  "dailyLimit": 3, // 每日限次
  "skin": "festive", // 皮肤（仅大转盘）
  "embed": false, // true=隐藏左侧配置面板
  "prizes": [
    { "name":"100元券", "emoji":"🎫", "prob":5, "color":"#F5C842", "type":"win" },
    { "name":"谢谢参与", "emoji":"😊", "prob":50, "color":"#6B7280", "type":"miss" }
  ]
}
🎰 各游戏 prizes 格式差异
游戏prob含义其他字段
大转盘整数% (0-100)stock库存
刮刮乐整数%，自动归一化score得分值
老虎机整数%→权重/2score=倍率
九宫格整数%，必须8个—
砸金蛋整数%desc描述
盲盒整数%按顺序→稀有度
🛠️ 手动生成链接（不用生成器）
// 在浏览器控制台执行
const cfg = {
  title: "我的活动",
  dailyLimit: 5,
  prizes: [
    {name:"大奖",emoji:"🏆",prob:5,type:"win"},
    {name:"谢谢",emoji:"😊",prob:95,type:"miss"}
  ]
};
const url = "https://games.jydigtal.com/wheel.html?c="
  + btoa(unescape(encodeURIComponent(JSON.stringify(cfg))));
console.log(url);
6. 游戏生成器使用流程 builder.html
零代码操作，3分钟生成一个活动链接
1
选择游戏类型

点击 6 种游戏类型卡片之一（大转盘/刮刮乐/老虎机/九宫格/砸金蛋/盲盒）。底部自动出现该游戏的使用提示。

2
填写基础配置

活动标题 / 副标题 / 每日次数。大转盘还可以选皮肤（紫/红/蓝/金）。

3
配置奖品

点 emoji 换图标，填写奖品名称、概率%、类型（中奖/未中）。右上角快速模板可一键套用电商/品牌/游戏/餐饮4套预设。概率条实时显示总计是否达到100%。

4
生成链接

点「🚀 生成游戏链接」，右侧显示完整 URL。可一键复制链接、新窗口预览、生成二维码。右侧 iframe 也在实时预览效果。

5
发给用户

把链接或二维码发给用户，他们直接打开浏览器就能玩，无需安装任何东西。每个链接的配置独立，可以针对不同活动生成不同链接。

⚠️ 注意事项：配置存在 URL 里，链接一旦发出配置就固定了。如果要修改奖品，需要重新在 builder 生成新链接再发给用户。
7. 经济系统 金币·经验·等级
驱动用户留存的核心机制
🪙 金币来源

· 游戏结束奖励：5~50/局
· 成就解锁：20~500/个
· 每日签到：20~100/天
· 商城购买（消耗）

⚡ 经验来源

· 每局游戏：10~50经验
· 等级公式：Lv.N 需要 N×100 经验
· 当前最高等级：无上限

🎖️ 50个成就

· 抽奖类：首次中奖、连续中奖等
· 游戏类：2048达到指定分、贪吃蛇闯关等
· 社交类：玩过10种游戏
· 解锁时自动弹 Toast + 发金币

🏪 商城可兑换内容（shop.html）

头像框（100~500金币）· 特效（200~800金币）· 优惠券道具（50~200金币）· 游戏称号（300~1000金币）
共20件商品，购买后存入本地 gh_shop，兑换记录可查账单。
